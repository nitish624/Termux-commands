 # JSON 
 - Datastore और Transpot
 - हल्की Data store करने के लिए
 - Curly Braces { }: "Object" darshate hain
 - Square Brackets [ ]: "Array" (list) ko darshate hain
 - Example
 {
  "name": "Rocky bhai",
  "isLoggedIn": true,
  "services": ["PAN", "Aadhar", "Voter"]
}

 # XML
 - Datastore और Transpot
 - खुद के बनाए हुए Tags जैसेः
 <ServiceCard>
    <Name>PAN Card</Name>
    <Status>Public</Status>
    <Price>107</Price>
</ServiceCard>

 - Android App Design: apps mein button kahan hoga, image ki size kya hogi, ye sab XML file mein likha jata hai.
 
 # YAML
 - YAML (.yaml ya .yml): Ye JSON jaisa hi hota hai lekin ismein brackets {} nahi hote. Ye padhne mein sabse aasaan hoti hai aur aksar "Settings" save karne ke liye use hoti hai.
 
 # CSV
 - Iska matlab hai Comma Separated Values. Ye Excel sheet jaisa hota hai. Agar aapke paas hazaron services ka data hai, toh CSV behtar hoti hai.
 
 # md
 - Markdown (.md): Ye documentation ke liye hoti hai. GitHub par README.md isi format mein hoti hai.
 
 # Java / Kt
 - Java/Kotlin (.java, .kt): Android app ka asli logic isi mein likha jata hai
 
 # PHP
 - PHP (.php): Agar aapko server par data save karna ho (jaise real database), toh PHP ka use hota hai.
 
 # SQL
 - SQL (.sql): Ye Database ki file hoti hai jahan user ka data (Mobile number, password) tables ke roop mein store hota hai
